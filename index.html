📅 Historical Period Hierarchies

Hierarchies change every ~3 months. Upload a hierarchy snapshot for a specific date range (e.g. Jan–Mar 2026) so historical TPH, CSAT, and IQS records are correctly attributed to the right supervisors and AMs. The system will immediately re-stamp all existing records in that range and will use this snapshot automatically for any future uploads covering the same period.

📅

Period Hierarchy

Columns: A.Manager, New Supervisor, Hire Date, NAME, C•U Email, ROLE

How it works

  1. Select the date range for this hierarchy snapshot (e.g. 2026-01-01 → 2026-03-31)
  2. Upload the hierarchy CSV that was valid during that period
  3. The system re-stamps all existing records in that range immediately
  4. Any future data uploads whose dates fall in this range will use this snapshot automatically

Overlap rule: if two periods overlap, the most recently uploaded snapshot takes priority for the overlapping dates.
